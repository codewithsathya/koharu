# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

Koharu is a local-first, ML-powered manga translator. A Rust workspace (detection → OCR → inpainting → LLM translation → text rendering) drives a Tauri desktop app with a Next.js (React 19) frontend. The same backend also serves a local HTTP/SSE API and an MCP server.

## Commands

Build/dev orchestration goes through `scripts/dev.ts` (auto-detects CUDA/MSVC and sets up the env). Use `bun run <script>`, not raw cargo, for app builds.

- `bun run dev` — Tauri dev (UI on port 4000, `release-with-debug` profile).
- `bun run build` — `tauri build --no-bundle`.
- `bun run cargo <args>` — cargo with the dev env applied (e.g. `bun run cargo build -p koharu-ml`).
- `bun run lint:ui` / `bun run test:ui` — UI oxlint / vitest.
- `bun run format` / `format:check` — oxfmt over `ui`, configs, `.vscode`.
- `cargo test --workspace --tests` — full Rust suite (unit + integration).
- Run one Rust test: `cargo test -p <crate> <test_name>` (e.g. `cargo test -p koharu-integration-tests`).
- Lint matches CI: `cargo fmt -- --check`, `cargo check`, `cargo clippy -- -D warnings`.

### Code generation (run after changing API or scene/protocol types)
- `bun run generate:openapi` — regenerate `ui/openapi.json` from `koharu-rpc` routes.
- `bun run generate:client` — regenerate the Rust integration-test client from the OpenAPI spec.
- `cd ui && bun run generate:api` — regenerate the TypeScript API client (orval) from `openapi.json`.

The OpenAPI spec is the contract between Rust handlers and both clients; keep it in sync when route signatures or DTOs change.

## Architecture

The backend is layered so types flow inward and I/O stays outward. Crate docs (`lib.rs` headers) are accurate and worth reading first.

- **koharu-core** — pure types, no I/O/tokio/HTTP. `scene` (project/pages/nodes), `op` (the `Op` enum — *the only way the scene mutates*, with patches + apply/inverse for undo), `blob` (`BlobRef` content hashes), `events` (`AppEvent`), `protocol` (non-scene DTOs), `style`/`font`. Everything else depends on this.
- **koharu-ml** — vision models (detection, OCR, inpainting, layout, font detection), each in its own submodule, running on candle. See README "ML Models" for the model list and HF sources.
- **koharu-llm** — LLM translation: local llama.cpp (`llama`, `sys`, `safe`), cloud providers under `providers/`, prompt/jinja templating, language detection, PaddleOCR-VL.
- **koharu-runtime** — runtime/dylib management: CUDA/ZLUDA/Vulkan setup, on-demand model + llama.cpp downloads, archive extraction, secret storage (system keychain).
- **koharu-ai** — Codex (ChatGPT) image-to-image provider.
- **koharu-renderer** — manga text rendering: OpenType shaping (harfrust/skrifa), script-aware line breaking, vertical CJK / RTL layout, glyph metrics, stroke/effect compositing.
- **koharu-psd** — layered PSD export (descriptor/engine_data/packbits writers).
- **koharu-app** — application layer wiring core + ml + llm + runtime together: top-level `App`/`AppSharedState`, `ProjectSession`, the engine `pipeline` (`Engine`/`Registry`/`PipelineSpec`), `history` (undo via op inverses), `BlobStore`, archive (project file) I/O, autosave, config, the event `bus`.
- **koharu-rpc** — transport over `koharu-app::App`: `routes/*` (one module per domain, typed axum handlers sharing `State<ApiState>`), `binary` (byte reads like `/scene.bin`, `/blobs/:hash`), `events` (SSE `GET /events`), `mcp` (per-route MCP tools via rmcp), `psd_export`. State is `Arc<BootstrapManager>`.
- **koharu** — the `default-members` binary: CLI (`--headless`, `--port`, `--host`, `--cpu`, `--download`, `--debug`), Tauri app bootstrap, panic/sentry/tracing, Windows shims.

State change flow: clients (UI or MCP) call rpc routes → `App` applies an `Op` to the `Scene` → the op produces an inverse patch (recorded in history) → an `AppEvent` is published on the bus → streamed to clients via SSE.

### Frontend (`ui/`)
Next.js 16 + React 19 (React Compiler), Tailwind v4, Radix UI, zustand + xstate for state, TanStack Query, i18next. `ui/openapi.json` + orval generate the typed API client; `ui/app/` holds routes/stores/io, `ui/lib/` holds events/ops/backend glue. Talks to the Rust backend over the same HTTP/SSE API.

## Conventions

- Rust edition 2024, GPL-3.0. Workspace deps are pinned centrally in root `Cargo.toml` — add/bump versions there, reference with `{ workspace = true }`.
- candle is patched to `mayocream/candle` forks (flash-attn / CUDA dynamic loading) via `[patch.crates-io]`; don't replace with upstream.
- Models, llama.cpp dylibs, and GPU runtimes download on demand at runtime — not vendored.
- Integration tests cache a shared llama.cpp runtime under `tests/integration-tests/.cache`; secret-service tests need a keyring (gnome-keyring on Linux CI).
- CHANGELOG is generated by git-cliff (`cliff.toml`); don't edit by hand.
- Per the repo AI policy (CONTRIBUTING.md): disclose AI usage and fully review/test AI-generated changes before submitting.
